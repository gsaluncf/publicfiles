Lab 6: Exploring Semi-Structured JSON Data

Objective: Learn to create and query tables with JSON data using dot notation syntax and the FLATTEN function.

Scenario: Snowbear Air is collecting JSON data. You're tasked with learning about semi-structured data and teaching other analysts how to work with it.

üìö Learning Objectives

  • Create a simple table with JSON data stored in VARIANT columns
  • Query JSON data using dot notation syntax
  • Query nested JSON data with dot notation
  • Use the FLATTEN function to query complex JSON structures
  • Determine and cast data types in query results

1.2 Create Simple JSON Table

Create a table with customer data where the info column contains JSON:

CREATE OR REPLACE TABLE customers AS
SELECT
    $1 AS id,
    parse_json($2) AS info
FROM
    VALUES
    (12712555, '{"name": {"first": "John", "last":"Smith"}}'),
    (98127771, '{"name": {"first": "Jane", "last":"Doe"}}');

üí° Understanding VARIANT

The VARIANT data type can store semi-structured data like JSON, Avro, ORC, Parquet, or XML. The parse_json() function converts a string to a VARIANT containing JSON.

1.3 View the Table Structure

SELECT * FROM customers;

Click on a row in the info column to display its JSON structure. You'll see the top-level key "name" contains an object with "first" and "last" keys.

2. Dot Notation Syntax

2.1 Extract JSON Data with Dot Notation

Use dot notation to extract values from JSON:

SELECT
    id,
    info:name.first AS first_name,
    info:name.last AS last_name
FROM
    customers;

üìñ Dot Notation Syntax

<column>:<level1_element>.<level2_element>.<level3_element>

  • Use a colon : after the column name
  • Use dots . to navigate through nested levels
  • Example: info:name.first navigates to column ‚Üí name object ‚Üí first property

2.2 Check Data Types

Notice the extracted values are in double quotes - they're still VARIANT type:

DESCRIBE RESULT last_query_id();

2.3 Cast to VARCHAR

Cast VARIANT to standard SQL types for better performance:

SELECT
    id,
    info:name.first::VARCHAR AS first_name,
    info:name.last::VARCHAR AS last_name
FROM
    customers;

-- Verify the data types
DESCRIBE RESULT last_query_id();

Now the columns are VARCHAR. Use ::VARCHAR (or other types) to cast VARIANT data for:

  • Better query performance
  • String functions and sorting
  • Comparisons and aggregations

2.4 Create Structured Table

CREATE OR REPLACE TABLE customers_structured AS
SELECT
    id,
    info:name.first::VARCHAR AS first_name,
    info:name.last::VARCHAR AS last_name
FROM
    customers;

SELECT * FROM customers_structured;

3. Nested JSON Data

3.1 Create More Complex JSON

Work with deeper nested structures and arrays:

CREATE OR REPLACE TABLE customers_nested AS
SELECT
    $1 AS id,
    parse_json($2) AS info
FROM
    VALUES
    (12712555, '{
        "name": {"first": "John", "last":"Smith"},
        "contact": {
            "address": {"street": "123 Main St", "city": "Seattle", "state": "WA"},
            "phones": ["206-555-1234", "206-555-5678"]
        }
    }'),
    (98127771, '{
        "name": {"first": "Jane", "last":"Doe"},
        "contact": {
            "address": {"street": "456 Pine Ave", "city": "Boston", "state": "MA"},
            "phones": ["617-555-9999"]
        }
    }');

3.2 Query Nested Objects

Navigate through multiple levels:

SELECT
    id,
    info:name.first::VARCHAR AS first_name,
    info:contact.address.city::VARCHAR AS city,
    info:contact.address.state::VARCHAR AS state
FROM
    customers_nested;

3.3 Query Array Elements

Access specific array elements using bracket notation:

SELECT
    id,
    info:name.first::VARCHAR AS first_name,
    info:contact.phones[0]::VARCHAR AS phone1,
    info:contact.phones[1]::VARCHAR AS phone2
FROM
    customers_nested;

‚ö†Ô∏è Array Index

Arrays are zero-indexed: [0] is the first element, [1] is the second, etc. If an index doesn't exist, the result is NULL.

4. FLATTEN Function

When you don't know how many array elements exist, or need to process all elements, use FLATTEN.

4.1 Basic FLATTEN Usage

FLATTEN explodes an array into multiple rows:

SELECT
    id,
    info:name.first::VARCHAR AS first_name,
    f.value::VARCHAR AS phone
FROM
    customers_nested,
    LATERAL FLATTEN(input => info:contact.phones) f;

This produces one row per phone number. Customer 12712555 has 2 phones ‚Üí 2 rows. Customer 98127771 has 1 phone ‚Üí 1 row.

üí° FLATTEN Output Columns

  • VALUE: The element from the array
  • INDEX: Position in the array (0, 1, 2...)
  • KEY: For objects, the key name
  • PATH: Full path to the element

4.2 FLATTEN with Object Arrays

If your array contains objects (not just strings), FLATTEN can help extract fields:

-- Example: orders array with objects
CREATE OR REPLACE TABLE customers_orders AS
SELECT
    $1 AS id,
    parse_json($2) AS info
FROM
    VALUES
    (12712555, '{
        "name": "John Smith",
        "orders": [
            {"order_id": 101, "amount": 250.00},
            {"order_id": 102, "amount": 175.50}
        ]
    }');

-- Flatten the orders array
SELECT
    id,
    info:name::VARCHAR AS customer_name,
    f.value:order_id::INT AS order_id,
    f.value:amount::NUMBER(10,2) AS amount
FROM
    customers_orders,
    LATERAL FLATTEN(input => info:orders) f;

4.3 Recursive FLATTEN

For deeply nested arrays, use recursive => TRUE:

-- This flattens all nested levels
SELECT
    f.value
FROM
    customers_nested,
    LATERAL FLATTEN(input => info, recursive => TRUE) f;

5. Practical Example

5.1 Complete Scenario

Extract all customer contact information including all phone numbers:

SELECT
    id,
    info:name.first::VARCHAR || ' ' || info:name.last::VARCHAR AS full_name,
    info:contact.address.street::VARCHAR AS street,
    info:contact.address.city::VARCHAR AS city,
    info:contact.address.state::VARCHAR AS state,
    f.value::VARCHAR AS phone_number,
    f.index AS phone_index
FROM
    customers_nested,
    LATERAL FLATTEN(input => info:contact.phones) f
ORDER BY id, phone_index;

6. Key Takeaways

  • VARIANT columns can store JSON and other semi-structured data
  • Use parse_json() to convert strings to JSON VARIANT
  • Dot notation syntax: column:level1.level2.level3
  • Array elements accessed with brackets: column:array[0]
  • Cast VARIANT to SQL types (::VARCHAR, ::INT, etc.) for better performance
  • FLATTEN explodes arrays into multiple rows for processing
  • LATERAL FLATTEN allows you to join flattened results with original table
  • Use recursive => TRUE for deeply nested structures
  • Check data types with DESCRIBE RESULT last_query_id()
