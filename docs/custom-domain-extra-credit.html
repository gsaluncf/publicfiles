Extra Credit: Register Your Own Domain

Overview

This extra credit assignment walks you through purchasing your own domain name and pointing it to your GitHub Pages site. By the end, your site will be accessible at a custom URL like yourname.com instead of username.github.io.

Why do this? Owning a domain is a professional skill. It demonstrates understanding of DNS, web infrastructure, and gives you a personal brand for your portfolio.

Cost Notice

Domain registration typically costs $10-20 per year depending on the domain extension (.com, .dev, .io, etc.). This is optional extra credit, so only proceed if you are comfortable with this expense.

Part 1: Configure GitHub Pages for Your Custom Domain

Before purchasing a domain, you need to tell GitHub what domain name you plan to use. This video walks through the process:

1Open Repository Settings

Navigate to your GitHub repository, click Settings in the top navigation bar.

2Go to Pages Section

In the left sidebar, click Pages (under "Code and automation").

3Enter Your Custom Domain

In the "Custom domain" field, enter the domain name you want to use (e.g., yourdomain.com). Click Save.

Important: Note the A Records

Click the "Learn more about configuring custom domains" link, which takes you to:

GitHub Custom Domain Documentation

You will need these A Records (IP addresses) when configuring your domain's DNS:

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

Write these down or keep this page open. You will need them in Part 2.

Part 2: Purchase and Configure Your Domain

This video demonstrates purchasing a domain from Squarespace Domains and configuring the DNS records:

1Choose a Domain Registrar

Go to domains.squarespace.com (or another registrar of your choice like Namecheap, Google Domains, Cloudflare, etc.).

2Search for Your Domain

Enter the domain name you want and check if it is available. Consider using your name, a project name, or something professional.

3Purchase the Domain

Create an account (or log in) and complete the purchase. A 1-year registration is typically sufficient to start.

4Access DNS Settings

After purchasing, navigate to your domain's management page. Look for Domains then DNS or DNS Settings.

5Remove Preset DNS Records

Most registrars add default records. Remove any existing A records or CNAME records that might conflict with your GitHub Pages configuration.

6Add A Records for the Root Domain

Add four A records pointing to GitHub's servers. These handle traffic to your "naked" domain (e.g., yourdomain.com without www):

What is the Root Domain?

The root domain (also called "apex domain" or "naked domain") is your domain without any subdomain prefix. For example:

  • yourdomain.com is the root domain
  • www.yourdomain.com is a subdomain

A records point the root domain to specific IP addresses. In this case, GitHub's servers.

Type Host/Name Value/Points To
A @ (or leave blank) 185.199.108.153
A @ (or leave blank) 185.199.109.153
A @ (or leave blank) 185.199.110.153
A @ (or leave blank) 185.199.111.153

The @ symbol represents the root domain in DNS configuration.

7Add a CNAME Record for www

Add a CNAME record so that www.yourdomain.com also works:

Type Host/Name Value/Points To
CNAME www yourusername.github.io

Replace yourusername with your actual GitHub username.

What is a CNAME Record?

A CNAME (Canonical Name) record creates an alias from one domain to another. Instead of pointing to an IP address, it points to another domain name.

When someone visits www.yourdomain.com, the CNAME tells their browser to look up yourusername.github.io instead.

Part 3: Wait for DNS Propagation and SSL

1Return to GitHub Pages Settings

Go back to your repository's Settings, then Pages. You should see "DNS Check in Progress" next to your custom domain.

2Wait for DNS Verification

DNS changes can take anywhere from a few minutes to 48 hours to propagate globally. Typically, it takes 5-30 minutes.

Be Patient

The video shows a common mistake: checking too early. Wait until GitHub shows "DNS check successful" before testing your domain.

3Wait for SSL Certificate

After DNS verification succeeds, GitHub will automatically provision an SSL certificate for HTTPS. This can take an additional 15-30 minutes.

Once complete, you will see a checkbox option to "Enforce HTTPS". Enable this for security.

4Test Your Domain

Visit your custom domain in a browser. You should see your GitHub Pages site with the index.html (or README.md) from your repository's root directory.

Troubleshooting

"DNS check unsuccessful"

Double-check that you entered the A records correctly. Ensure there are no typos in the IP addresses. Wait a bit longer for propagation.

"Certificate not yet generated"

SSL provisioning happens after DNS verification. Wait 15-30 minutes after DNS check succeeds.

Site shows wrong content or 404

Ensure your repository has an index.html file in the root (or the folder you configured for GitHub Pages). The README.md will also work if no index.html exists.

www works but root domain does not (or vice versa)

Verify you have both the A records (for root) and the CNAME record (for www) configured correctly.

Summary: What You Learned

DNS and Domain Concepts

  • Domain Registration: Purchasing the rights to use a domain name
  • A Records: DNS entries that map a domain to IPv4 addresses
  • CNAME Records: DNS entries that create aliases between domain names
  • Root/Apex Domain: The base domain without subdomains (e.g., example.com)
  • DNS Propagation: The time it takes for DNS changes to spread globally
  • SSL/TLS Certificates: Enable HTTPS for secure, encrypted connections

Distributed Systems Connection

DNS itself is one of the largest distributed systems in the world. When you configure DNS records, you are interacting with a globally distributed database that routes billions of requests daily. Your domain configuration propagates across thousands of DNS servers worldwide.
