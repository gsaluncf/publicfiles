Q: What if my schema is different from the reference?

A: That's OK! As long as your schema:

  • Is in 3NF (properly normalized)
  • Has all required constraints (PK, FK, CHECK, UNIQUE)
  • Supports the required queries
  • Is well-documented

There's no single "correct" schema. Good database design involves trade-offs.

Q: How do I know if my schema is in 3NF?

A: Check these rules:

1NF (First Normal Form):

  • All values are atomic (no lists or arrays in a single cell)
  • No repeating groups (e.g., rating1, rating2, rating3 columns)

2NF (Second Normal Form):

  • Must be in 1NF
  • No partial dependencies (all non-key columns depend on the ENTIRE primary key)

3NF (Third Normal Form):

  • Must be in 2NF
  • No transitive dependencies (non-key columns don't depend on other non-key columns)

Example of 3NF violation:

If you have issuer_name and issuer_state in the bonds table, that's a transitive dependency (bond → issuer_id → issuer_name).

Solution: Move issuer info to separate issuers table.

Q: Why are there 2,139 ratings but only 2,000 bonds?

A: Some bonds have multiple ratings over time (rating changes). This is realistic:

  • 139 bonds (7%) have rating changes (upgrades or downgrades)
  • 1,861 bonds (93%) have only their initial rating

This is a key analytical opportunity - you can analyze rating migration!

Q: Why do only 1,671 bonds have trades? What about the other 329?

A: This is realistic! Not all bonds trade actively:

  • 1,671 bonds (84%) have at least one trade
  • 329 bonds (16%) have zero trades (illiquid)

This is a discovery opportunity - you can analyze liquidity patterns!

Q: The trades only go from 2020-2024, but bonds were issued from 2010-2024. Why?

A: We're simulating a 5-year trading window (2020-2024). This is realistic:

  • Older bonds (issued 2010-2019) may have traded before 2020
  • We only have trade data for the recent 5-year period
  • This is common in financial datasets (historical data is expensive)

For your queries: Focus on the 2020-2024 period where you have trade data.

Q: What's the difference between a correlated and non-correlated subquery?

A:

Non-correlated subquery (runs once):

SELECT * FROM bonds
WHERE coupon_rate > (SELECT AVG(coupon_rate) FROM bonds);

Correlated subquery (runs for each row):

SELECT * FROM bonds b1
WHERE coupon_rate > (
    SELECT AVG(coupon_rate)
    FROM bonds b2
    WHERE b2.issuer_id = b1.issuer_id  -- References outer query
);

Either type is acceptable for Query 3 (Week 11).

Q: What's a window function and why do I need it?

A: Window functions perform calculations across rows related to the current row, without collapsing them (like GROUP BY does).

Example - Moving Average:

SELECT
    bond_id,
    trade_date,
    trade_price,
    AVG(trade_price) OVER (
        PARTITION BY bond_id
        ORDER BY trade_date
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS moving_avg_3day
FROM trades;

This calculates a 3-day moving average for each bond without grouping the data.

Resources:

  • PostgreSQL tutorial: Window Functions
  • Sample Queries - Queries 3, 4, 8, 10

Q: What's a CTE (Common Table Expression)?

A: A CTE is a temporary named result set that you can reference within a query. Think of it as a "query variable."

Example:

WITH high_yield_bonds AS (
    SELECT bond_id, yield
    FROM trades
    WHERE yield > 4.0
)
SELECT b.bond_id, b.cusip, hyb.yield
FROM bonds b
JOIN high_yield_bonds hyb ON b.bond_id = hyb.bond_id;

Why use CTEs?

  • Makes complex queries more readable
  • Can reference the same subquery multiple times
  • Can chain multiple CTEs together

Q: How do I join trades with economic indicators? They don't have a direct relationship.

A: Join through state and date:

SELECT t.*, ei.unemployment_rate, ei.treasury_10yr
FROM trades t
JOIN bonds b ON t.bond_id = b.bond_id
JOIN issuers i ON b.issuer_id = i.issuer_id
JOIN economic_indicators ei ON
    i.state = ei.state AND
    DATE_TRUNC('month', t.trade_date) = ei.date;

Key: Economic indicators are monthly, so use DATE_TRUNC('month', ...) to match.

Q: What if my query returns too many rows?

A: Use LIMIT:

SELECT * FROM trades LIMIT 20;

For your submission, show the first 20 rows of results.

Q: How do I create a yield curve?

A:

1. Query: Get yield vs. time to maturity

SELECT
    EXTRACT(YEAR FROM (maturity_date - CURRENT_DATE)) AS years_to_maturity,
    AVG(yield) AS avg_yield,
    rating
FROM bonds b
JOIN trades t ON b.bond_id = t.bond_id
JOIN credit_ratings cr ON b.bond_id = cr.bond_id
GROUP BY years_to_maturity, rating;

2. Visualize: Line chart with:

  • X-axis: Years to maturity
  • Y-axis: Average yield
  • Color: Credit rating

Q: What does "overlay economic indicators" mean?

A: Create a chart with two Y-axes:

  • Right Y-axis: Economic indicator (unemployment, Treasury yield)
  • X-axis: Time (date)

This shows how bond metrics move with economic conditions.
