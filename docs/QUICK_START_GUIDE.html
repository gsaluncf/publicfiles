Quick Start Guide

Municipal Bond Database Project

Prerequisites: PostgreSQL installed, basic SQL knowledge

What This Guide Covers

This guide will help you:

  • Set up PostgreSQL and create a database
  • Understand the CSV file structure
  • Load data into simple tables (no relationships yet)
  • Explore the data to understand what you're working with

What you'll need to do yourself :

  • Design the proper schema with Primary Keys (PK)
  • Define Foreign Keys (FK) to create relationships between tables
  • Add CHECK constraints for data validation
  • Add UNIQUE constraints where appropriate
  • Create indexes for performance
  • Normalize to 3NF

Step 1: Download the Project

  1. Download or clone the project repository
  2. Navigate to the project directory
  3. Verify you have all the CSV files in the data/final/ folder

You should see these 6 files:

  • issuers.csv
  • bond_purposes.csv
  • bonds.csv
  • credit_ratings.csv
  • trades.csv
  • economic_indicators.csv

Step 2: Create the Database

Option A: Using Command Line

# Create database
createdb municipal_bonds

# Connect to it
psql municipal_bonds

Option B: Using pgAdmin

  1. Open pgAdmin
  2. Right-click "Databases" → "Create" → "Database"
  3. Name: municipal_bonds
  4. Click "Save"

Option C: Using DataGrip

  1. Open DataGrip
  2. Click the "+" icon → "Data Source" → "PostgreSQL"
  3. Enter your PostgreSQL connection details:
    • Host: localhost
    • Port: 5432
    • Database: postgres (default)
    • User: your PostgreSQL username
    • Password: your PostgreSQL password
  4. Click "Test Connection" to verify
  5. Click "OK"
  6. Right-click the data source → "New" → "Database"
  7. Name: municipal_bonds
  8. Click "OK"
  9. Open a query console (right-click database → "New" → "Query Console")

Step 3: Create Simple Tables for Exploration

Important: These are basic tables with NO relationships, constraints, or indexes. This is just to load the data so you can explore it.

  • Primary Keys (PK)
  • Foreign Keys (FK)
  • CHECK constraints
  • UNIQUE constraints
  • Indexes
  • Proper normalization (3NF)

Create basic tables to hold the CSV data:

-- Simple table for issuers (no constraints yet)
CREATE TABLE issuers (
    issuer_id INTEGER,
    issuer_name TEXT,
    state TEXT,
    issuer_type TEXT,
    population INTEGER,
    tax_base_millions NUMERIC
);

-- Simple table for bond purposes
CREATE TABLE bond_purposes (
    purpose_id INTEGER,
    purpose_category TEXT,
    purpose_description TEXT
);

-- Simple table for bonds
CREATE TABLE bonds (
    bond_id INTEGER,
    issuer_id INTEGER,
    purpose_id INTEGER,
    cusip TEXT,
    bond_type TEXT,
    coupon_rate NUMERIC,
    issue_date DATE,
    maturity_date DATE,
    face_value NUMERIC,
    duration NUMERIC,
    tax_status TEXT
);

-- Simple table for credit ratings
CREATE TABLE credit_ratings (
    rating_id INTEGER,
    bond_id INTEGER,
    rating_agency TEXT,
    rating TEXT,
    rating_date DATE,
    outlook TEXT
);

-- Simple table for trades
CREATE TABLE trades (
    bond_id INTEGER,
    trade_date DATE,
    trade_price NUMERIC,
    yield NUMERIC,
    quantity INTEGER,
    buyer_type TEXT
);

-- Simple table for economic indicators
CREATE TABLE economic_indicators (
    state TEXT,
    date DATE,
    unemployment_rate NUMERIC,
    treasury_10yr NUMERIC,
    treasury_20yr NUMERIC,
    vix_index NUMERIC
);

Step 6: Explore the Data

Now that the data is loaded, explore it to understand what you're working with:

Look at the Data

-- See what's in each table
SELECT * FROM issuers LIMIT 10;
SELECT * FROM bonds LIMIT 10;
SELECT * FROM trades LIMIT 10;

-- Count rows
SELECT COUNT(*) FROM bonds;
SELECT COUNT(*) FROM trades;

Understand Relationships

Look at the data and ask yourself:

  • What connects bonds to issuers? (Hint: look at the columns)
  • What connects trades to bonds?
  • Can one bond have multiple trades? Multiple ratings?
  • What should be the Primary Key for each table?
  • What Foreign Keys do you need?

Try a Simple JOIN

-- Even without foreign keys, you can join on matching columns
SELECT
    b.bond_id,
    b.cusip,
    i.issuer_name,
    i.state
FROM bonds b
JOIN issuers i ON b.issuer_id = i.issuer_id
LIMIT 20;

Notice: This works even without FK constraints, but constraints would ensure data integrity!

Step 7: Understand What's Missing

These Simple Tables Are Missing:

  • Primary Keys: No unique identifiers enforced
  • Foreign Keys: No referential integrity between tables
  • CHECK Constraints: No validation (e.g., coupon_rate > 0)
  • UNIQUE Constraints: No uniqueness enforcement (e.g., CUSIP should be unique)
  • NOT NULL Constraints: Critical fields can be null
  • Indexes: Queries will be slow on large datasets
  • Normalization: May have redundancy issues

Check Current Table Structure

-- List all tables
\dt

-- Describe a table (notice: no constraints!)
\d bonds

You'll see the columns but NO primary keys, foreign keys, or constraints. That's what you'll add in Week 9!

Common Issues & Solutions

Issue 1: "Permission denied" when loading CSV

Solution: Use absolute paths in \COPY commands

-- Windows
\COPY issuers FROM 'C:/full/path/to/issuers.csv' CSV HEADER;

-- Mac/Linux
\COPY issuers FROM '/full/path/to/issuers.csv' CSV HEADER;

Issue 2: "Relation does not exist"

Solution: Make sure you created the tables first (Step 3)

Issue 3: Date format errors

Solution: Dates should be in YYYY-MM-DD format. The CSV files are already in this format, so this shouldn't be an issue.

Issue 4: Column count mismatch

Solution: Make sure your CREATE TABLE statements match the CSV file structure exactly. Check the CSV headers!

Quick Reference

Common SQL Commands

-- List tables
\dt

-- Describe table
\d table_name

-- View data
SELECT * FROM table_name LIMIT 10;

-- Count rows
SELECT COUNT(*) FROM table_name;

-- Exit psql
\q

Common Aggregations

-- Count
SELECT COUNT(*) FROM bonds;

-- Average
SELECT AVG(coupon_rate) FROM bonds;

-- Min/Max
SELECT MIN(trade_price), MAX(trade_price) FROM trades;

-- Group by
SELECT state, COUNT(*)
FROM issuers
GROUP BY state;

Common Joins

-- Inner join
SELECT *
FROM bonds b
JOIN issuers i ON b.issuer_id = i.issuer_id;

-- Left join
SELECT *
FROM bonds b
LEFT JOIN trades t ON b.bond_id = t.bond_id;

-- Multiple joins
SELECT *
FROM bonds b
JOIN issuers i ON b.issuer_id = i.issuer_id
JOIN bond_purposes bp ON b.purpose_id = bp.purpose_id;
