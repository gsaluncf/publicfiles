Continuation: Load data from external stages and explore validation and error handling strategies.

3. Load Data from External Stage

3.1 Explore the Stage

Review Stage Properties:

DESCRIBE STAGE TRAINING_DB.TRAININGLAB.ED_STAGE;

Review File Contents:

SELECT n.$1
FROM @training_db.traininglab.ed_stage/load/lab_files/region.tbl n;

‚ö†Ô∏è Format Mismatch

Note: The stage's file format uses comma delimiters, but our file uses pipe delimiters. We'll override the stage's format by specifying our MYPIPEFORMAT in the COPY command.

List Files in Stage:

LIST @training_db.traininglab.ed_stage/load/lab_files/ pattern='.*region.*';

3.2 Load Uncompressed File

Load region.tbl:

COPY INTO REGION
FROM @training_db.traininglab.ed_stage/load/lab_files/
FILES = ('region.tbl')
FILE_FORMAT = (FORMAT_NAME = MYPIPEFORMAT);

Verify Data:

SELECT * FROM REGION;

4. Load Compressed File

4.1 Prepare for GZIP Load

Clear Table:

TRUNCATE TABLE region;

Confirm GZIP File Exists:

LIST @training_db.traininglab.ed_stage/load/lab_files/ pattern='.*region.*';

4.2 Load GZIP File

Load region.tbl.gz:

COPY INTO region
FROM @training_db.traininglab.ed_stage/load/lab_files/
FILES = ('region.tbl.gz')
FILE_FORMAT = ( FORMAT_NAME = MYGZIPPIPEFORMAT);

Verify and Clean Up:

SELECT * FROM region;

-- Resize and suspend warehouse
ALTER WAREHOUSE WOODCHUCK_WH SET WAREHOUSE_SIZE=XSmall;
ALTER WAREHOUSE WOODCHUCK_WH SUSPEND;
