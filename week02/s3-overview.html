AWS S3 Overview

Amazon S3 (Simple Storage Service) is AWS's object storage service — a massive, globally distributed file system in the cloud. This page explains what S3 is, why we use it for static websites, how much it costs, and what alternatives exist.

What is AWS S3?

Amazon S3 is object storage designed to store and retrieve any amount of data from anywhere on the web. Think of it as a massive hard drive in the cloud that you can access via HTTP/HTTPS.

Key Characteristics

  • Store any type of file: HTML, CSS, JavaScript, images, videos, data files, backups, etc.
  • Access via URLs: Every object gets a unique URL for HTTP/HTTPS access
  • Pay-as-you-go: Only pay for storage used + requests made + data transferred
  • 99.999999999% durability: That's 11 nines — your files won't get lost
  • Automatic scaling: No capacity planning needed — S3 grows with your needs
  • Global availability: Choose from 30+ AWS regions worldwide

Why Use S3 for Static Websites?

Traditional Web Hosting Problems

  • Server management: Need to install updates, patch security vulnerabilities, configure web servers
  • Fixed costs: Pay for server capacity even when your site has zero visitors
  • Scaling complexity: Traffic spike? Your server crashes or you overpay for unused capacity
  • Deployment friction: FTP uploads, SSH access, file permissions, restart services

S3 Static Website Hosting Advantages

  • Zero server management: AWS handles all infrastructure, updates, security
  • Automatic scaling: Handle 10 visitors or 10 million — S3 scales automatically
  • Pennies per month: Small sites cost less than $0.01/month
  • Built-in redundancy: Files automatically replicated across multiple data centers
  • Simple deployment: Just upload files — no server configuration needed
  • Perfect for: Portfolios, documentation, landing pages, marketing sites, SPAs

What Does "Static" Mean?

Static website: HTML, CSS, and JavaScript files served directly to the browser without server-side processing.

What you CAN do:

  • Display content (text, images, videos)
  • Interactive JavaScript (animations, form validation, API calls)
  • Single Page Applications (React, Vue, Angular)
  • Call external APIs (weather, maps, payment processors)

What you CANNOT do:

  • Run server-side code (PHP, Python, Ruby, Node.js)
  • Direct database access (MySQL, PostgreSQL)
  • Server-side authentication/sessions
  • File uploads to the server

Note: You can work around these limitations using serverless functions (AWS Lambda) and APIs, but that's beyond this course.

How Much Does S3 Cost?

S3 pricing has three components: storage, requests, and data transfer. Here's the breakdown for us-east-1 region (2026 pricing):

Storage Costs

S3 Standard Storage: $0.023 per GB per month

Examples:

  • 10 KB starter site: $0.00000023/month (essentially free)
  • 1 MB portfolio site: $0.000023/month
  • 100 MB site with images: $0.0023/month (~$0.03/year)
  • 1 GB site with videos: $0.023/month (~$0.28/year)

Request Costs

  • PUT/POST requests: $0.005 per 1,000 requests (uploading files)
  • GET requests: $0.0004 per 1,000 requests (page views)

Examples:

  • 100 page views: $0.00004
  • 1,000 page views: $0.0004
  • 10,000 page views: $0.004

Data Transfer Costs

  • First 100 GB/month: FREE
  • After 100 GB: $0.09 per GB

Examples:

  • 1,000 page views × 1 MB each = 1 GB = FREE
  • 10,000 page views × 1 MB each = 10 GB = FREE
  • 100,000 page views × 1 MB each = 100 GB = FREE

AWS Free Tier

For the first 12 months after creating your AWS account, you get:

  • 5 GB of S3 Standard storage
  • 20,000 GET requests
  • 2,000 PUT requests
  • 100 GB data transfer out

What "More" Looks Like

This assignment teaches basic S3 static website hosting. Here's what the progression looks like as you build more sophisticated systems:

Level 1: Basic S3 (This Assignment)

  • Upload files manually or via CLI
  • HTTP-only access (no HTTPS)
  • S3 website endpoint URL (long, ugly: bucket-name.s3-website-us-east-1.amazonaws.com)
  • No custom domain
  • Cost: Pennies per month

Level 2: S3 + CloudFront (Optional Extra Credit)

CloudFront = AWS's Content Delivery Network (CDN)

  • Caches content at edge locations worldwide (faster load times)
  • HTTPS support with free SSL/TLS certificates
  • Custom domain support (your-site.com)
  • Better security (DDoS protection)
  • Cost: ~$0.085 per GB transferred (first 10 TB)

See CloudFront Optional Extra Credit for details.

Level 3: S3 + CloudFront + Route 53 (Production)

Route 53 = AWS's DNS service

  • Manage custom domains (buy or transfer domains)
  • Advanced routing (geo-location, failover, weighted routing)
  • Health checks and monitoring
  • Integration with other AWS services
  • Cost: $0.50/month per hosted zone + $0.40 per million queries

Level 4: Full Static Site Pipeline (Professional)

  • S3 for storage
  • CloudFront for global delivery
  • Route 53 for DNS
  • AWS Certificate Manager for free SSL certificates
  • CodePipeline for automated deployments (CI/CD)
  • Lambda@Edge for dynamic content at edge locations
  • Cost: ~$5-20/month depending on traffic

Level 5: Hybrid Architecture (Advanced)

"Serverless" full-stack application

  • S3 for static assets (HTML, CSS, JS, images)
  • API Gateway + Lambda for backend logic
  • DynamoDB for database
  • Cognito for user authentication
  • No servers to manage — everything scales automatically
  • Cost: Pay per request, scales to zero when idle

Why Learn S3 When Free Alternatives Exist?

Great question! Here's why S3 is worth learning even though GitHub Pages, Netlify, and Vercel are free:

1. Industry Standard

  • AWS has 32% cloud market share (2026)
  • Most companies use AWS in some capacity
  • S3 skills transfer to other AWS services

2. Foundation for Cloud Computing

  • Understanding S3 teaches core cloud concepts (object storage, regions, availability zones)
  • Object storage is fundamental to modern architecture
  • AWS CLI skills apply to other cloud providers (Azure CLI, gcloud)

3. Integration Ecosystem

  • S3 integrates with 200+ AWS services
  • Can build complex systems starting with S3 (data lakes, backup systems, content delivery)
  • Learn how cloud services work together

4. Career Skills

  • AWS certifications are valuable (Solutions Architect, Developer, SysOps)
  • Cloud skills in high demand
  • Understanding costs/billing is critical for real-world projects

5. Real-World Experience

  • Free tiers don't teach cost management
  • Learning to clean up resources is important (prevents surprise bills)
  • Understanding pricing helps make architecture decisions

Bottom line: GitHub Pages is great for personal projects. But if you want to work in cloud computing, DevOps, or backend engineering, understanding AWS S3 is essential.
