Lab: Deploy a Static Website to AWS S3

This lab guides you through deploying a static website to Amazon S3 using both AWS Console UI and AWS CLI methods. You'll learn the complete workflow from bucket creation to public website deployment.

Time estimate: 45-60 minutes

Important: Two Methods Provided

This lab provides two complete methods for deploying your S3 website:

  • Method 1: AWS CLI Commands (Recommended) ‚Äî Direct AWS commands that work on Windows, Mac, and Linux (faster, repeatable, industry-standard)

Recommendation: Use Method 1 (CLI) to learn the commands professionals use. The Console UI (Method 2) is available if you prefer a visual approach or want to understand what the CLI commands are doing behind the scenes.

Method 1: AWS CLI Commands (Recommended)

This method uses AWS CLI commands that work on Windows, Mac, and Linux. These are the same commands used by professionals in production environments.

Why Use CLI Commands?

  • Cross-platform: Same commands work on Windows, Mac, and Linux
  • Faster: Complete deployment in minutes instead of clicking through multiple screens
  • Repeatable: Easy to tear down and redeploy if needed
  • Industry-standard: Learn the tools used in real DevOps workflows
  • Automation-ready: Commands can be saved in scripts for later use

Step 0: Set Your Bucket Name

First, choose a globally unique bucket name. We'll use it throughout the commands below.

Bucket naming rules:

  • Format: ncf-s26-yourname-site
  • Example: ncf-s26-jsmith-portfolio
  • Must be lowercase, no spaces, globally unique across all AWS

üí° Tip: Replace YOUR-BUCKET-NAME in all commands below with your actual bucket name.

Step 1: Verify AWS CLI Setup

Before starting, verify your AWS CLI is configured correctly:

aws sts get-caller-identity

You should see your AWS account ID and user ARN. If this fails, run aws configure to set up your credentials.

Step 2: Create S3 Bucket

Run this command to create your S3 bucket in the us-east-1 region:

aws s3api create-bucket --bucket YOUR-BUCKET-NAME --region us-east-1

Example:

aws s3api create-bucket --bucket ncf-s26-jsmith-portfolio --region us-east-1

‚úÖ Success looks like: You'll see JSON output with the bucket location.

Step 3: Enable Static Website Hosting

Configure the bucket to serve as a static website:

aws s3 website s3://YOUR-BUCKET-NAME/ --index-document index.html --error-document index.html

What this command does

  • --index-document index.html ‚Äî Sets the default page
  • --error-document index.html ‚Äî Sets the error page (useful for single-page apps)

Step 4: Upload Website Files

Navigate to your repository's site directory and sync the files to S3:

# Navigate to your repository (adjust path as needed)
cd path/to/your/repository

# Sync the site files to S3
aws s3 sync ./site s3://YOUR-BUCKET-NAME --delete

What this command does

  • sync ‚Äî Uploads files and keeps them in sync (only uploads changed files)
  • ./site ‚Äî Source directory (your local files)
  • s3://YOUR-BUCKET-NAME ‚Äî Destination (your S3 bucket)
  • --delete ‚Äî Removes files from S3 that don't exist locally (keeps bucket clean)

‚úÖ Success looks like: You'll see output showing upload: site/index.html to s3://... for each file.

Step 5: Disable Block Public Access

‚ö†Ô∏è Security Warning

By default, AWS blocks all public access to S3 buckets for security. To host a public website, you must disable this protection. Only do this for buckets intended to be public websites.

Run this command to allow public access:

aws s3api put-public-access-block \
  --bucket YOUR-BUCKET-NAME \
  --public-access-block-configuration "BlockPublicAcls=false,IgnorePublicAcls=false,BlockPublicPolicy=false,RestrictPublicBuckets=false"

Windows users: Remove the backslash and put the command on one line:

aws s3api put-public-access-block --bucket YOUR-BUCKET-NAME --public-access-block-configuration "BlockPublicAcls=false,IgnorePublicAcls=false,BlockPublicPolicy=false,RestrictPublicBuckets=false"

Step 6: Add Bucket Policy for Public Read Access

Disabling Block Public Access isn't enough ‚Äî you must also add a bucket policy that explicitly grants public read permissions.

First, create a file called bucket-policy.json with this content (replace YOUR-BUCKET-NAME):

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
    }
  ]
}

Then apply the policy:

aws s3api put-bucket-policy --bucket YOUR-BUCKET-NAME --policy file://bucket-policy.json

Understanding the Bucket Policy

  • "Principal": "*" ‚Äî Allows anyone (public access)
  • "Action": "s3:GetObject" ‚Äî Allows reading objects (not writing or deleting)
  • "Resource": ".../*" ‚Äî Applies to all objects in the bucket

Step 7: Get Your Website URL

Your website URL follows this format:

http://YOUR-BUCKET-NAME.s3-website-us-east-1.amazonaws.com

Example: http://ncf-s26-jsmith-portfolio.s3-website-us-east-1.amazonaws.com

Open this URL in your browser to see your live website!

‚úÖ Success!

If you see your website, congratulations! You've successfully deployed a static website to AWS S3 using the CLI.

Next steps:

  • Customize the site files in site/
  • Re-upload with: aws s3 sync ./site s3://YOUR-BUCKET-NAME --delete
  • Take a screenshot for your Canvas submission

Updating Your Website

After making changes to your site files, re-sync them to S3:

aws s3 sync ./site s3://YOUR-BUCKET-NAME --delete

The sync command is smart ‚Äî it only uploads files that have changed!

Teardown (IMPORTANT for Cost Management)

When you're done with the assignment and it's been graded, delete all resources to avoid charges:

# Delete all files in the bucket
aws s3 rm s3://YOUR-BUCKET-NAME --recursive

# Delete the bucket
aws s3api delete-bucket --bucket YOUR-BUCKET-NAME --region us-east-1

‚ö†Ô∏è Cost Management

CRITICAL: You must delete your bucket after the assignment is graded. S3 charges for storage and requests. While costs are minimal (typically $0.01-0.10), leaving resources running is bad practice.

Method 2: AWS Console UI (Alternative)

This method uses the AWS web interface for visual learners who prefer clicking through the console instead of using CLI commands. Note: Method 1 (CLI) is recommended for faster iterations and professional workflows.

üìö Official AWS Tutorial

For detailed screenshots and step-by-step guidance, refer to the official AWS documentation:

AWS Tutorial: Configuring a static website on Amazon S3

The steps below provide a condensed version of this tutorial.

Console UI Deployment Steps

Step 1: Create S3 Bucket

  1. Sign in to the AWS S3 Console
  2. Click "Create bucket"
  3. Enter a globally unique bucket name (e.g., ncf-s26-yourname-site)
  4. Select AWS Region: us-east-1 (or your preferred region)
  5. Leave other settings as default
  6. Click "Create bucket"

Step 2: Enable Static Website Hosting

  1. Click on your newly created bucket name
  2. Go to the "Properties" tab
  3. Scroll down to "Static website hosting" section
  4. Click "Edit"
  5. Select "Enable"
  6. Set Index document: index.html
  7. Set Error document: index.html
  8. Click "Save changes"
  9. Note the website endpoint URL displayed (you'll need this later)

Step 3: Upload Website Files

  1. Go to the "Objects" tab
  2. Click "Upload"
  3. Click "Add files" and select all files from your site/ directory (index.html, styles.css, script.js)
  4. Click "Upload"
  5. Wait for upload to complete, then click "Close"

Step 4: Disable Block Public Access

‚ö†Ô∏è Security Warning

This step makes your bucket publicly accessible. Only do this for buckets intended to host public websites.

  1. Go to the "Permissions" tab
  2. Find "Block public access (bucket settings)"
  3. Click "Edit"
  4. Uncheck "Block all public access"
  5. Click "Save changes"
  6. Type confirm in the confirmation dialog
  7. Click "Confirm"

Step 5: Add Bucket Policy for Public Read Access

  1. Still in the "Permissions" tab, scroll to "Bucket policy"
  2. Click "Edit"
  3. Paste the following policy (replace YOUR-BUCKET-NAME with your actual bucket name):
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
    }
  ]
}
  1. Click "Save changes"

Step 6: Test Your Website

  1. Go back to the "Properties" tab
  2. Scroll to "Static website hosting"
  3. Click the Bucket website endpoint URL
  4. Your website should load in a new tab!

‚úÖ Deployment Complete!

Your website is now live! The URL format is:

http://your-bucket-name.s3-website-us-east-1.amazonaws.com

Copy this URL for your Canvas submission!

Updating Your Website (Console Method)

To update files after making changes:

  1. Edit files locally in your site/ directory
  2. Go to S3 Console ‚Üí Your bucket ‚Üí Objects tab
  3. Select the file(s) to update and click "Delete"
  4. Click "Upload" and re-upload the updated files
  5. Refresh your browser to see changes

üí° Recommendation: Switch to CLI for Updates

Uploading files through the console is tedious for frequent updates. Consider using Method 1 (CLI) for faster iterations:

aws s3 sync ./site s3://YOUR-BUCKET-NAME --delete

This single command uploads only changed files and removes deleted ones ‚Äî much faster than manual uploads!

Teardown (Console Method)

‚ö†Ô∏è Delete Resources After Grading

To avoid ongoing charges, delete your bucket after the assignment is graded:

  1. Go to S3 Console ‚Üí Your bucket
  2. Click "Empty" button
  3. Type permanently delete and confirm
  4. Go back to S3 Console bucket list
  5. Select your bucket and click "Delete"
  6. Type your bucket name to confirm deletion

Or use the CLI teardown script for faster cleanup: .\99_teardown.bat

Optional: Automation Scripts

Want to save the CLI commands as reusable scripts? Here are templates for different operating systems.

üí° Why Create Scripts?

  • Save time when updating your website
  • Avoid typing the same commands repeatedly
  • Learn automation and DevOps practices
  • Easy to share with teammates or reuse for other projects

For Windows (PowerShell)

Create a file called deploy.ps1:

# Set your bucket name
$BUCKET_NAME = "ncf-s26-yourname-site"

# Sync files to S3
aws s3 sync ./site s3://$BUCKET_NAME --delete

Write-Host "Website updated! Visit: http://$BUCKET_NAME.s3-website-us-east-1.amazonaws.com"

Run with: .\deploy.ps1

For Mac/Linux (Bash)

Create a file called deploy.sh:

#!/bin/bash

# Set your bucket name
BUCKET_NAME="ncf-s26-yourname-site"

# Sync files to S3
aws s3 sync ./site s3://$BUCKET_NAME --delete

echo "Website updated! Visit: http://$BUCKET_NAME.s3-website-us-east-1.amazonaws.com"

Make it executable and run:

chmod +x deploy.sh
./deploy.sh

For Windows (Batch File)

Create a file called deploy.bat:

@echo off
REM Set your bucket name
set BUCKET_NAME=ncf-s26-yourname-site

REM Sync files to S3
aws s3 sync .\site s3://%BUCKET_NAME% --delete

echo Website updated! Visit: http://%BUCKET_NAME%.s3-website-us-east-1.amazonaws.com
pause

Run with: deploy.bat
