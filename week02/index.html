Week 2 Overview (Tue Feb 3, Thu Feb 5)

This week introduces cloud object storage and static website hosting using Amazon S3. You will deploy a live website to AWS, configure public access policies, and learn both AWS Console and CLI workflows. Optional extra credit includes CloudFront CDN integration and custom domain configuration.

Tuesday — AWS S3 Fundamentals

  • Concepts: Object storage, buckets, static website hosting, bucket policies, public vs private access.
  • AWS S3 overview: What it is, why it matters, cost structure, alternatives (GitHub Pages, Netlify, Vercel).
  • Practice: Create S3 bucket via Console UI, enable static website hosting, upload files.

Thursday — Deployment Automation & Best Practices

  • AWS CLI for S3: aws s3 sync, bucket policies, automation scripts.
  • Cost management: Understanding S3 pricing, setting up billing alerts, teardown procedures.
  • Optional: CloudFront CDN, HTTPS, custom domains (continuation of Week 1 extra credit).

Learning Objectives

By the end of this week, you will be able to:

  • Explain what AWS S3 is and when to use it for static website hosting
  • Create and configure S3 buckets using both AWS Console and CLI
  • Enable static website hosting and configure index/error documents
  • Manage bucket policies and public access settings
  • Deploy and update website files using aws s3 sync
  • Understand S3 pricing model and implement cost management practices
  • Properly tear down AWS resources to avoid unnecessary charges
  • Compare S3 with alternative hosting solutions (GitHub Pages, Netlify, Vercel)
  • Optionally integrate CloudFront CDN for HTTPS and custom domains

Important: Cost Management

This assignment uses real AWS resources that incur charges. However, costs are minimal:

  • Expected cost: Less than $0.01 if resources are deleted within one week
  • S3 storage: ~$0.0002/month for starter site (~10 KB)
  • S3 requests: ~$0.0004 per 1,000 page views
  • Data transfer: First 100 GB/month is FREE

CRITICAL: After your assignment is graded, run the 99_teardown.bat script to delete all resources. Failure to do so may result in ongoing charges.
