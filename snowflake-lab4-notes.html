Lab 4: Loading Structured Data

Objective: Learn to load data from external stages into Snowflake tables using COPY INTO commands, file formats, and error handling.

Scenario: As a data engineer at Snowbear Air, create and populate a REGION table from pre-existing files in external stages.

  • Handle loading errors with different strategies

2. Create Table and File Formats

2.1 Create REGION Table

Create the target table for loading region data:

CREATE OR REPLACE TABLE REGION (
       R_REGIONKEY NUMBER(38,0) NOT NULL,
       R_NAME      VARCHAR(25)  NOT NULL,
       R_COMMENT   VARCHAR(152)
);

2.2 Create File Formats

Pipe-Delimited Format:

CREATE OR REPLACE FILE FORMAT MYPIPEFORMAT
  TYPE = CSV
  COMPRESSION = NONE
  FIELD_DELIMITER = '|'
  FILE_EXTENSION = 'tbl'
  ERROR_ON_COLUMN_COUNT_MISMATCH = FALSE;

GZIP Pipe-Delimited Format:

CREATE OR REPLACE FILE FORMAT MYGZIPPIPEFORMAT
 TYPE = CSV
 COMPRESSION = GZIP
 FIELD_DELIMITER = '|'
 FILE_EXTENSION = 'tbl'
 ERROR_ON_COLUMN_COUNT_MISMATCH = FALSE;

Check your understanding

  • What's the difference between the two file formats?
  • Why do we set ERROR_ON_COLUMN_COUNT_MISMATCH = FALSE?
  • What file extensions are supported for each format?
